## 📖 О проекте (About the Project)

Данный репозиторий представляет собой финальную (восьмую) лабораторную работу по программированию. Проект эволюционировал из простого консольного приложения в **полноценную Enterprise-систему**. 

Это не просто программа, а демонстрация глубокого понимания архитектуры программного обеспечения, безопасного сетевого взаимодействия, многопоточности и современного проектирования UI/UX.

---

## ✨ Ключевые особенности (Key Features)

### 🎨 1. Современный интерфейс (Pure JavaFX)
- Интерфейс создан **программно (без FXML)** для максимальной гибкости и модульности.
- Стильный **Dark Mode** дизайн, вдохновленный современными библиотеками (например, `shadcn/ui`).
- Интерактивные диалоговые окна, всплывающие уведомления (Alerts) и встроенный терминал с подсветкой синтаксиса.

### ⚡ 2. Высокопроизводительный сервер (NIO & Multithreading)
- Сервер построен на неблокирующей архитектуре **Java NIO** (`Selector`, `SocketChannel`).
- Обработка запросов реализована через **асинхронный конвейер** с использованием пулов потоков:
  - `ForkJoinPool` — для чтения данных.
  - `FixedThreadPool` — для бизнес-логики и отправки ответов (через `CompletableFuture`).
- Защита памяти: использование потокобезопасной коллекции `ConcurrentSkipListSet` и строгая синхронизация составных операций.

### 🌍 3. Интернационализация "на лету" (i18n)
- Поддержка 4 локалей: **Русский, Словацкий, Албанский, Английский (Канада)**.
- Переключение языка происходит мгновенно без перезапуска приложения с использованием паттерна **EventBus**.
- Автоматическое форматирование чисел, дат и времени согласно выбранной локали.

### 🔐 4. База данных и Безопасность (JDBC)
- Полный отказ от файлового хранения в пользу реляционной СУБД **PostgreSQL**.
- Регистрация и авторизация пользователей с хэшированием паролей алгоритмом **SHA-384**.
- Защита от SQL-инъекций с помощью `PreparedStatement`.
- Строгое разграничение прав доступа: пользователи могут изменять/удалять только свои объекты.

### 📡 5. Живая синхронизация и Визуализация (Real-time & Canvas)
- **Фоновый поллинг (Auto-refresh):** UI автоматически обновляется каждые 3 секунды через `ScheduledExecutorService` и `Platform.runLater()`, синхронизируя данные между всеми подключенными клиентами.
- **Интерактивный Радар (Canvas):** Объекты отрисовываются на радаре с анимацией. Каждый владелец имеет свой уникальный цвет (защита от коллизий хэшей).
- **Живой поиск (Live Search):** Мгновенная фильтрация и сортировка таблицы с использованием **Stream API**.

## 🏗️ Архитектура и Паттерны проектирования

В проекте строго соблюдены принципы чистого кода (SOLID) и применены следующие паттерны проектирования:
*   **DTO (Data Transfer Object):** Классы `Request` и `Response` для обмена данными.
*   **Command:** Инкапсуляция серверных и клиентских операций.
*   **Repository:** Изоляция логики работы с базой данных PostgreSQL.
*   **Observer / EventBus:** Для обновления интерфейса при смене языка.
*   **Factory & Singleton:** Для создания и управления единственными экземплярами сервисов.
